Privileged Identity Management & Zero Standing Privileges

Concentrix Ltd. · Azure Entra ID / PIM
Why I needed this

A security audit revealed that 30+ service accounts and engineers held permanent Global Admin or Owner roles in Azure — a massive blast radius if compromised. Regulators flagged this as a critical finding under ISO 27001. I was tasked with eliminating standing privileges entirely.

  • Implemented Azure PIM (Privileged Identity Management) across all privileged roles — converting permanent assignments to time-bound, approval-gated, MFA-required Just-in-Time activations.
  • Configured Conditional Access policies enforcing compliant device, MFA, and named-location restrictions for all admin role activations.
  • Eliminated unused service principal permissions by right-sizing RBAC assignments using Azure Access Reviews — automated quarterly reviews with auto-revocation on no response.
  • Built Sentinel detection rules to alert on PIM role activations outside business hours, from unfamiliar locations, or without MFA completion.

Workload Identity Federation & Secrets Elimination

Star Compliance & Devi Technologies · Azure DevOps + AWS
Why I needed this

CI/CD pipelines were authenticating to Azure and AWS using long-lived client secrets stored in pipeline variables — a secret sprawl and rotation nightmare. A leaked secret had previously caused a near-miss incident. I needed to eliminate static secrets from pipelines entirely.

  • Replaced all Azure DevOps service principal client secrets with Workload Identity Federation (OIDC) — pipelines now authenticate using short-lived tokens, no secrets stored anywhere.
  • Implemented AWS IAM Roles Anywhere and OIDC federation for GitHub Actions — GitHub workflows assume IAM roles directly without AWS access keys.
  • Integrated HashiCorp Vault for dynamic secret generation — database credentials, API keys, and certificates issued per-pipeline-run with automatic expiry.
  • Audited and rotated all legacy static credentials; implemented automated credential rotation using AWS Secrets Manager and Azure Key Vault rotation policies.

Kubernetes Security Hardening & Runtime Protection

Concentrix Ltd. & Star Compliance · AKS / EKS
Why I needed this

A client's AKS workload had no admission control — developers were deploying containers as root, using images with critical CVEs, and with no network policies isolating namespaces. A container escape vulnerability in the wild made this immediately urgent. I had to harden the cluster without disrupting running workloads.

  • Deployed Azure Policy for AKS (Gatekeeper-based admission control) — blocked deployment of privileged containers, enforced read-only root filesystems, and required resource limits on all pods.
  • Implemented Kubernetes Network Policies to enforce namespace isolation — services could only communicate via explicitly defined ingress/egress rules, blocking lateral movement paths.
  • Configured Defender for Containers with runtime threat detection — alerted on suspicious process execution inside containers (reverse shells, crypto miners, privilege escalation attempts).
  • Enforced image scanning in ACR using Trivy and Defender for Containers — admission webhook blocked deployment of images with CRITICAL CVEs above a defined threshold.
  • Applied CIS Kubernetes Benchmark (L1 & L2) hardening across AKS control plane and node pools — API server audit logging, etcd encryption, and kubelet authentication enforced.

Detection Rule Building & Fine-Tuning

Concentrix Ltd. · Sentinel + Cortex XSOAR
Why I needed this

The SOC was drowning in 3,000+ daily alerts — 90% false positives. Analysts were experiencing alert fatigue and missing real incidents hidden in the noise. I needed to rebuild the detection layer from scratch with quality over quantity as the guiding principle.

  • Built 40+ MITRE ATT&CK mapped detection rules in Sentinel (KQL) and Cortex XSOAR (XQL) — each rule tied to a specific TTP with documented rationale, test cases, and tuning history.
  • Applied UEBA baselines to replace static threshold rules — "login from unusual country" rules now fired relative to each user's 30-day baseline, eliminating 80% of false positives from that category.
  • Built watchlist-driven suppression logic — dynamic allow-lists (IT admin IPs, service accounts, maintenance windows) injected into rules via Sentinel Watchlists, context-enriching every alert.
  • Implemented detection-as-code — Sentinel analytic rules stored as YAML in Azure DevOps, deployed via pipeline with peer review, version control, and automated rollback on regression.
  • Tuned Defender ASR rules in audit mode for 2 weeks, analysed telemetry to identify legitimate business processes blocked, then enforced block mode with targeted exclusions.Alert noise ↓75%
