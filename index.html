Multi-Cloud Posture Management (CSPM/CNAPP)

Concentrix & Devi Technologies · Azure + AWS
Why I needed this

Client environments had sprawling Azure and AWS deployments with no centralised visibility into misconfigurations. No one could answer "are we compliant?" without manual checks across hundreds of resources.

  • Deployed Defender for Cloud (CSPM) — Secure Score tracking, regulatory compliance dashboards (PCI DSS, CIS), and automated recommendations across all subscriptions.
  • Connected AWS Security Hub aggregating GuardDuty, Inspector, and Macie findings alongside Azure Defender in a single pane of glass.
  • Built custom Azure Policy initiatives enforcing tagging, encryption, and NSG rules — blocking non-compliant deployments at provision time.Misconfigs ↓40%
  • Used AWS Control Tower with SCPs to enforce organisation-wide guardrails preventing overly permissive IAM policies.
🔐

Privileged Identity Management & Zero Standing Privileges

Concentrix Ltd. · Azure Entra ID / PIM
Why I needed this

A security audit revealed 30+ engineers held permanent Global Admin or Owner roles — a critical ISO 27001 finding. I was tasked with eliminating standing privileges entirely.

  • Implemented Azure PIM — converting permanent assignments to time-bound, approval-gated, MFA-required Just-in-Time activations.
  • Configured Conditional Access policies enforcing compliant device, MFA, and named-location restrictions for all admin role activations.
  • Eliminated unused permissions via Azure Access Reviews — automated quarterly reviews with auto-revocation.
  • Built Sentinel detection rules alerting on PIM activations outside business hours, unfamiliar locations, or without MFA.
🔑

Workload Identity Federation & Secrets Elimination

Star Compliance · Azure DevOps + AWS
Why I needed this

CI/CD pipelines were authenticating using long-lived client secrets stored in pipeline variables. A leaked secret had caused a near-miss incident — I needed to eliminate static secrets entirely.

  • Replaced all Azure DevOps service principal secrets with Workload Identity Federation (OIDC) — pipelines authenticate using short-lived tokens, no secrets stored.
  • Implemented AWS IAM Roles Anywhere and OIDC federation for GitHub Actions — assume IAM roles directly without AWS access keys.
  • Integrated HashiCorp Vault for dynamic secret generation — database credentials and API keys issued per-pipeline-run with automatic expiry.
🐙

No automated security gates — vulnerabilities only found during quarterly pen tests. A developer accidentally committed an AWS secret to a public repo which was immediately exploited. Security needed to be in the developer workflow.

  • Dependabot with grouped PRs and auto-merge for patch updates — reduced dependency remediation time from weeks to days.

Kubernetes Security Hardening & Runtime Protection

Concentrix & Star Compliance · AKS / EKS
Why I needed this

AKS workload had no admission control — developers deploying containers as root, using images with critical CVEs. A container escape vulnerability in the wild made this immediately urgent.

  • Deployed Azure Policy for AKS (Gatekeeper) — blocked privileged containers, enforced read-only root filesystems and resource limits.
  • Implemented Kubernetes Network Policies enforcing namespace isolation — blocking lateral movement paths between services.
  • Configured Defender for Containers runtime detection — alerting on reverse shells, crypto miners, and privilege escalation inside containers.
  • Enforced CIS Kubernetes Benchmark (L1 & L2) — API server audit logging, etcd encryption, and kubelet authentication enforced.

Detection Rule Building & Fine-Tuning

SOC receiving 3,000+ daily alerts — 90% false positives. Analysts experiencing alert fatigue and missing real incidents. Needed to rebuild the detection layer with quality over quantity.

Specialist Capability
Log Engineering & Detection Fine-Tuning

I own the full detection pipeline — from raw log source through custom collection, DCR-based ingestion, ASIM normalisation, ingestion cost optimisation, and KQL detection rule fine-tuning.

📥 Custom Log Collection & DCRs

  • Deployed Azure Monitor DCRs to ingest custom logs from on-premises servers, custom apps, syslog endpoints, and third-party appliances into Log Analytics Workspaces.
  • Configured AMA (replacing legacy MMA) across hybrid endpoints managing DCR associations at scale.
  • Built custom ingestion pipelines using Logs Ingestion API and DCR transformations for sources without native connectors.
  • Created custom Log Analytics tables (Basic & Analytics tiers) with selective retention for cost vs performance balance.
  • Used WEF with custom XPath queries within DCRs collecting only high-value security events.

🔄 Log Normalisation & ASIM

  • Implemented ASIM normalisation across Sentinel — standardising logs from firewalls, DNS, auth, network, and process events.
  • Authored custom ASIM parsers (KQL functions) for Cisco ASA, Palo Alto, CrowdStrike, and Netskope.
  • Applied DCR-based transformations at ingestion time to parse, filter, and enrich fields before landing in workspace.
  • Used Regex-based parsing in Splunk and QRadar to extract structured fields from raw syslog and CEF data.

⚙️ Ingestion Fine-Tuning & Cost Optimisation

  • Performed ingestion volume analysis using KQL (Usage table, _BilledSize) — top 5 high-volume sources accounted for 60% of costs.
  • Implemented DCR event filtering (XPath + KQL stream declarations) dropping non-security events before workspace ingestion.
  • Routed high-volume raw flows to Basic Logs and detection-critical data to Analytics tier — significant cost reduction without coverage loss.
  • Automated ingestion monitoring with KQL workbooks and alert rules flagging anomalous ingestion spikes.

🎯 Detection Rule Development & Fine-Tuning

  • Built 40+ MITRE ATT&CK-mapped detection rules in Sentinel (KQL) and Cortex (XQL) — covering credential access, lateral movement, defence evasion, and exfiltration TTPs.
  • Applied UEBA baselines and watchlist-driven suppression logic for context-aware alert enrichment.
  • Tuned ASR rules in audit mode, analysed telemetry, then enforced block mode with targeted exclusions.
  • Managed Sentinel analytic rules as YAML/JSON via Azure DevOps — detection-as-code with version control and automated deployment.
