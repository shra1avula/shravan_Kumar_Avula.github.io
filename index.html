Specialist Capability
Log Engineering & Detection Fine-Tuning

A core part of my detection engineering practice is building the log infrastructure that feeds it — from raw source to actionable, high-fidelity alerts. I own the full pipeline: custom log collection → DCR-based ingestion → normalisation → query optimisation → detection rule tuning.

📥 Custom Log Collection & Data Collection Rules (DCR)

  • Designed and deployed Azure Monitor Data Collection Rules (DCRs) to ingest custom logs from non-native sources including on-premises servers, custom applications, syslog endpoints, and third-party security appliances into Log Analytics Workspaces.
  • Configured the Azure Monitor Agent (AMA) as a replacement for the legacy MMA, managing DCR associations across hybrid endpoints (Azure VMs, Arc-enabled servers, on-prem Linux/Windows).
  • Built custom log ingestion pipelines using the Logs Ingestion API and DCR transformations to ingest structured/unstructured data from sources not natively supported by Sentinel connectors.
  • Created custom tables in Log Analytics Workspace (Basic and Analytics tiers) and mapped ingestion streams to appropriate retention tiers to optimise cost vs. query performance.
  • Integrated Windows Event Forwarding (WEF) and custom XPath queries within DCRs to selectively collect high-value security events, reducing noise at the collection layer.

🔄 Log Normalisation & ASIM (Advanced SIEM Information Model)

  • Implemented ASIM normalisation schemas across Sentinel to standardise log formats from heterogeneous sources (firewall, DNS, authentication, network, process events) into a unified query layer.
  • Authored custom ASIM parsers using KQL functions to normalise vendor-specific log formats (Cisco ASA, Palo Alto, CrowdStrike, Netskope) into standard ASIM event schemas.
  • Applied DCR-based transformations at ingestion time using KQL to parse, filter, and enrich fields before data lands in the workspace — reducing post-ingestion processing overhead.
  • Used Regex-based log parsing within Splunk and QRadar to extract structured fields from raw syslog and CEF-formatted data, enabling consistent field mapping across SIEM platforms.
  • Maintained parser versioning and documentation, enabling SOC analysts to query across multi-source data without knowledge of vendor-specific field names.

⚙️ Log Ingestion Fine-Tuning & Cost Optimisation

  • Performed ingestion volume analysis using KQL (Usage table, _BilledSize) to identify high-volume, low-value log sources and implemented targeted filtering at the DCR transformation layer to reduce ingestion costs.
  • Configured event filtering within DCRs using XPath and KQL stream declarations to drop noisy, non-security-relevant events (e.g., verbose IIS logs, low-risk audit events) before they reach the workspace.
  • Applied Basic Logs tier for high-volume, infrequently queried data (e.g., raw network flows, verbose debug logs) and Analytics tier for detection-critical data, achieving significant cost reduction without losing coverage.
  • Implemented data capping and sampling strategies on development/test environment log sources to prevent accidental over-ingestion during pipeline testing.
  • Automated ingestion monitoring with KQL-based workbooks and alert rules to flag anomalous ingestion spikes, enabling proactive cost and capacity management.

🎯 Detection Rule Development & Fine-Tuning

  • Built 40+ high-fidelity detection rules in Microsoft Sentinel (KQL) and Cortex XSOAR (XQL), mapped to MITRE ATT&CK TTPs — covering techniques including credential access, lateral movement, defence evasion, and exfiltration.
  • Performed iterative false positive analysis by reviewing alert history, entity behaviour baselines, and environment-specific patterns — refining rule logic using exclusion lists, dynamic thresholds, and conditional filters to improve signal-to-noise ratio.
  • Applied entity-based analytics and UEBA (User and Entity Behaviour Analytics) within Sentinel to detect anomalous behaviours that static threshold rules would miss, improving detection of insider threats and compromised accounts.
  • Tuned ASR (Attack Surface Reduction) rules in Defender for Endpoint — balancing security posture with operational productivity by analysing audit mode telemetry before enforcing block mode rules.
  • Designed watchlist-driven detection logic in Sentinel — using dynamic allow-lists (known admin IPs, service accounts, maintenance windows) to context-enrich alerts and suppress benign positives.
  • Built detection-as-code workflows by managing Sentinel analytic rules as YAML/JSON via Azure DevOps pipelines — enabling version control, peer review, and automated deployment of detection content.
  • Created custom TTP libraries aligned to MITRE ATT&CK, enabling systematic coverage gap analysis and prioritised rule development roadmaps based on threat intelligence.
